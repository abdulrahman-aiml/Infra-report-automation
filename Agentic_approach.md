# 📄 Agentic AI–Driven AWS Instance Health Classification & Reporting System

---

## 1. 📌 Problem Statement

The organization manages approximately **2000 AWS EC2 instances** and requires a **monthly health report** categorizing instances into:

* 🟢 Green (Optimal)
* 🟡 Amber (Moderate)
* 🔴 Red (Critical)

### Current Process:

* Data exported from ServiceNow (Excel/CSV)
* Manual classification and report preparation

### Challenges:

* Time-consuming manual effort
* Inconsistent classification
* Lack of intelligent insights
* No scalability for growing infrastructure

---

## 2. 🎯 Objective

To design an **intelligent, automated reporting system** that combines:

* Deterministic data processing (for accuracy)
* Agentic AI (for insights and decision support)

---

## 3. 🧠 Solution Overview (Agentic AI Approach)

The solution follows a **Hybrid Agentic Architecture**, where:

* A **rule-based system** ensures accurate classification
* An **AI agent layer** adds intelligence, insights, and recommendations

---

## 4. 🏗️ Architecture Components

### Core AWS Services:

* Amazon S3 → File storage
* AWS Lambda → Data processing
* Amazon Bedrock → AI agent execution
* Amazon SNS → Notifications
* Amazon CloudWatch → Logging & monitoring

---

## 5. 🔄 End-to-End Workflow

### Step 1: File Ingestion

* User uploads ServiceNow export file to S3

### Step 2: Automated Processing

* Lambda function is triggered
* Performs:

  * Data cleaning
  * Validation
  * Rule-based classification

### Step 3: Report Generation

* Excel file generated with:

  * Detailed classification
  * Summary metrics

### Step 4: Agentic AI Processing

* Summary data passed to AI agent via Bedrock
* Agent performs:

  * Trend analysis
  * Risk identification
  * Recommendation generation

### Step 5: Output Delivery

* Final report + AI insights stored in S3
* Notification sent to stakeholders

---

## 6. ⚙️ Deterministic Classification Engine

### Purpose:

Ensure **accuracy, consistency, and auditability**

### Example Rules:

| Category | CPU Usage | Memory Usage |
| -------- | --------- | ------------ |
| 🟢 Green | < 50%     | < 60%        |
| 🟡 Amber | 50–75%    | 60–80%       |
| 🔴 Red   | > 75%     | > 80%        |

> These rules are configurable based on business requirements.

---

## 7. 🤖 Agentic AI Layer

### Powered By:

* Amazon Bedrock

---

### Role of AI Agents:

#### 1. Insight Generation

* “15% of instances are in critical condition”
* “Amber instances increased compared to previous cycle”

#### 2. Root Cause Analysis (Basic)

* Identify patterns across:

  * Instance types
  * Regions
  * Environments

#### 3. Recommendations

* Scale up/down suggestions
* Resource optimization advice
* Risk mitigation steps

#### 4. Natural Language Reporting

* Converts raw data into **executive-friendly summaries**

---

## 8. 🧩 Agent Design (Conceptual)

### Agent Inputs:

* Summary statistics
* Top critical instances
* Historical data (optional)

### Agent Outputs:

* Text-based insights
* Recommendations
* Alerts

---

### Example Agent Output:

> “Out of 2000 instances, 15% are in the Red category, primarily concentrated in production workloads. Immediate scaling is recommended for high CPU utilization instances. Additionally, 20% of Green instances are underutilized and may be optimized for cost reduction.”

### Sheet 2: Summary

* Total counts (Green / Amber / Red)

### Sheet 3: AI Insights

* Generated by agent
